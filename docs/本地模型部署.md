# 本地开源模型部署（可微调）

目标：在本机部署小参数量开源模型（GGUF），并接入 `Weather Smarter`。

## 推荐路线（LM Studio + LoRA）
这是当前最稳妥的路径：**LoRA 微调 → 转 GGUF → LM Studio 推理 → OpenAI 兼容接入**。

1. 用 `training/train_lora.py` 做 LoRA/QLoRA 微调（HF 模型）
2. 将 LoRA 合并或转换为 GGUF
3. 在 LM Studio 中加载 GGUF
4. 通过 OpenAI 兼容接口接入本项目

## 具体下载清单（我已替你选好）
1. 训练基座（HF）：`Qwen/Qwen3-0.6B`
2. 推理模型（GGUF）：`Qwen3-0.6B` 的 **Q4_K_M** 量化 GGUF（社区仓库，稳定可用）

## 下载方式（可二选一）
方式 A：LM Studio 内直接搜索并下载  
- 关键词：`Qwen3-0.6B` + `GGUF` + `Q4_K_M`

方式 B：使用 `huggingface-cli` 下载
```
pip install huggingface_hub
huggingface-cli download Qwen/Qwen3-0.6B --local-dir E:\models\qwen3-0.6b
huggingface-cli download MaziyarPanahi/Qwen3-0.6B-GGUF Qwen3-0.6B.Q4_K_M.gguf --local-dir E:\models
```

## 1. 安装依赖
```
pip install -r requirements-local-llm.txt
```

## 2. 准备模型
下载 GGUF 模型并放到本地磁盘，例如：
```
E:\models\qwen2.5-1.5b-instruct-q4_k_m.gguf
```

## 3. 配置环境变量（cmd）
```
setx LLM_PROVIDER llama_cpp
setx LLM_MODEL_PATH "E:\models\qwen2.5-1.5b-instruct-q4_k_m.gguf"
setx LLM_GPU_LAYERS 20
setx LLM_CTX 2048
```
说明：
- `LLM_GPU_LAYERS`：GPU 加速层数，显存不够就调小
- `LLM_CTX`：上下文长度

## 4. 启动服务
```
py -m uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

## 5. 验证
```
curl -X POST http://127.0.0.1:8000/api/v1/query ^
  -H "Content-Type: application/json" ^
  -d "{\"query\":\"今天适合跑步吗\"}"
```
成功返回 `decision` 且无 `LLM_NOT_CONFIGURED` 即可。

## 6. 可微调（LoRA / QLoRA）
使用 `training/train_lora.py` 训练 LoRA 适配器，见：
`training/README.md`

**LM Studio 使用提示**
- LM Studio 仅支持 **GGUF 推理**，不支持训练。  
- LoRA 训练后需要 **合并或转换为 GGUF** 才能在 LM Studio 中加载。

若需要在 `llama_cpp` 中使用 LoRA：
- 将 LoRA 适配器转换为 GGUF
- 配置：
```
setx LLM_LORA_PATH "E:\models\adapter.gguf"
```

## 7. LM Studio（OpenAI 兼容）
如果模型在 LM Studio 中运行：
```
setx LLM_PROVIDER openai
setx OPENAI_BASE_URL http://127.0.0.1:1234/v1
setx OPENAI_COMPAT_MODE chat
setx OPENAI_API_KEY lm-studio
setx LLM_MODEL "你的模型名称"
```

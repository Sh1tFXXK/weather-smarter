# 病痛知识微调指南（Health-Fine-Tuning）

目标：为健康能力提供“可控、可解释、非医疗”的语言与知识增强，避免给出诊断/用药建议。

## 一、范围与边界
- 允许：健康风险提醒、行为建议、生活方式建议。
- 禁止：诊断、用药、治疗方案、急救指导（除非明确来自权威公共指南且以“建议就医”结尾）。
- 输出必须包含非医疗免责声明：**“仅供参考，非医疗建议。”**

## 二、数据与知识来源（建议优先级）
1. 权威公共指南或机构科普（卫健委/疾控/WHO 等）
2. 医院/学会发布的健康宣教
3. 公开且可溯源的科普文章（需要标注来源）

## 三、健康触发器（示例）
用于规则与 LLM 的共享知识片段：
- 风湿/关节不适：
  - 触发条件：高湿、低温、降雨持续、风力偏大
  - 建议关键词：保暖、防潮、关节热敷、避免久坐潮湿环境
- 哮喘/呼吸敏感：
  - 触发条件：AQI 偏高、风沙大、温差剧烈
  - 建议关键词：减少户外高强度活动、佩戴口罩、室内通风
- 过敏性鼻炎：
  - 触发条件：花粉季、风大、温差大
  - 建议关键词：减少暴露、清洗鼻腔、保持室内清洁
- 光敏/皮肤敏感：
  - 触发条件：UV 强、长时间日晒
  - 建议关键词：防晒霜、帽子、避开正午

## 四、数据格式建议（用于知识库/微调）
推荐 JSONL 格式：
```json
{"id":"rheumatism_humidity","title":"潮湿与关节不适","content":"潮湿寒冷天气可能加重关节不适，建议注意保暖、防潮。","tags":["health","rheumatism"],"domain":"health"}
```

## 五、微调策略
- 微调目标不是生成医学知识，而是“表达更自然、更符合场景、避免越界”。
- 建议使用少量高质量样本，优先覆盖“天气→健康提醒”的场景。
- 样本中必须包含“风险理由 + 行动建议 + 免责声明”三段结构。

## 六、提示词约束（适用于 LLM 增强）
- 输出 JSON 且固定字段：`advice`, `riskLevel`, `reasons`, `actions`
- 禁止输出诊断、用药、治疗方案。
- 必须包含免责声明：`"仅供参考，非医疗建议。"`

## 七、验收标准
- 输出内容包含风险理由与行动建议。
- 不含医疗诊断或用药建议。
- 在高风险场景提示“建议咨询医生或专业机构”。

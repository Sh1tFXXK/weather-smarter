# 面向生活决策的自然语言交互式天气服务系统—开发文档

本项目旨在以自然语言交互帮助用户完成与天气相关的生活决策（通勤、运动、出行、穿衣、洗车、紫外线防护等），并通过多源数据与个性化记忆提供可解释建议与主动提醒。

## 一、系统概述
- 系统名称：面向生活决策的自然语言交互式天气服务系统（Weather Smarter）
- 系统目标：结合天气、空气质量、日程与用户偏好，生成个性化、可解释的生活建议，支持语音交互与 3D 助手小人界面。
- 关键技术：
  - LLM Agent（自然语言理解、多场景决策）
  - RAG 知识库（规则与经验增强）
  - Memory 模块（用户行为与偏好）
  - 多源数据接口（天气、AQI、地图/日程）
  - 语音交互（ASR + TTS）
  - 前端助手小人 3D 界面（动作、表情、气泡提示）

## 二、系统模块说明
| 模块 | 功能 | 技术/接口 |
| --- | --- | --- |
| 用户交互界面 | 语音/文本输入，小人动作表情与气泡提示 | Web 前端 + WebGL/Three.js/Unity WebGL |
| ASR 语音识别 | 语音转文本 | Whisper / Web Speech API |
| TTS 语音合成 | 文本转语音输出 | Coqui TTS / Edge/Azure TTS |
| LLM Agent | 意图识别、任务理解、决策规划、主动提醒 | LangChain/LlamaIndex/自建模型 |
| RAG 知识库 | 生活规则、天气经验、模板检索 | 向量库（FAISS/pgvector/ES） |
| 用户 Memory | 存储用户行为、偏好与上下文 | SQLite/MySQL/向量库 |
| 多源数据接口 | 天气/AQI/地理编码/日程 | OpenWeatherMap、高德地图、Calendar API |
| 多场景决策 | 规则引擎 + LLM 推理 | 业务规则 + 阈值/打分模型 |
| 主动提醒 | 基于事件与预测的提醒 | Agent + Memory + 调度器 |
| 小人动作控制 | 与文本/语音同步动作表情 | 3D 动画控制（Three.js/Unity） |

## 三、数据流程
1) 用户交互：
   - 说话 → ASR 转文本；或直接输入文字。
   - 系统可观察用户行为（晨跑、上下班时段、日程）。
2) 后端处理：
   - LLM Agent 解析意图与场景 → 信息补齐（时间/地点/活动）。
   - 拉取多源数据（天气、AQI、日程、偏好）。
   - 检索 RAG 规则与经验；读取 Memory（历史行为/偏好）。
   - 多场景决策模块综合计算风险与建议。
3) 输出：
   - 文本建议 + 证据/理由；TTS 播报；小人动作表情同步。
   - 根据天气或日程变化触发主动提醒。

## 四、数据库设计（E-R 概念）
1) 用户表 `User`

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| user_id | INT PK | 用户 ID |
| name | VARCHAR | 用户姓名 |
| preferences | JSON | 偏好（运动、饮食、穿衣等） |
| created_at | TIMESTAMP | 创建时间 |

2) 行为记录表 `UserBehavior`

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| behavior_id | INT PK | 行为记录 ID |
| user_id | INT FK | 用户 ID |
| activity_type | VARCHAR | 活动类型（晨跑/出行/运动等） |
| timestamp | DATETIME | 记录时间 |
| weather_context | JSON | 天气上下文 |

3) 天气数据表 `WeatherData`

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| weather_id | INT PK | 天气记录 ID |
| location | VARCHAR | 地点 |
| date | DATE | 日期 |
| temperature | FLOAT | 温度 |
| humidity | FLOAT | 湿度 |
| rainfall | FLOAT | 降雨量 |
| wind_speed | FLOAT | 风速 |
| air_quality | VARCHAR | 空气质量 |

4) 任务表 `Task`

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| task_id | INT PK | 任务 ID |
| user_id | INT FK | 用户 ID |
| type | VARCHAR | 任务类型（出行/运动/家庭/健康） |
| scheduled_time | DATETIME | 计划时间 |
| status | VARCHAR | 完成状态 |
| priority | INT | 优先级 |

5) 知识库表 `Knowledge`

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| knowledge_id | INT PK | 知识 ID |
| scenario | VARCHAR | 场景（穿衣/出行/运动） |
| rule_text | TEXT | 规则或建议文本 |
| vector_repr | BLOB | 向量表示（RAG 检索） |

## 本地开发与运行（Python）
- 创建并激活虚拟环境：
  - Windows: `py -3 -m venv .venv && .\\.venv\\Scripts\\activate`
  - macOS/Linux: `python3 -m venv .venv && source .venv/bin/activate`
- 安装依赖：`pip install -r requirements.txt`
- 运行开发服务：`uvicorn main:app --reload`
- 环境配置：在 `config/config.yaml` 或 `.env` 中设置密钥与端点（不提交仓库）。
- 依赖建议（requirements.txt）：
  - `fastapi`, `uvicorn[standard]`, `pydantic`, `requests`
  - `sqlalchemy` 或数据库驱动, `alembic`（可选）
  - `faiss-cpu`/`chromadb`（RAG）, `langchain`/`llama-index`（可选）
  - `openai-whisper`/`whisperx`（ASR 可选）, `TTS`/`edge-tts`（TTS 可选）

## 测试与质量（Python）
- 测试：使用 `pytest`，运行 `pytest -q`；集成测试覆盖 `/api/v1/query` 与决策链路。
- 代码风格：`black` 格式化；`ruff` 静态检查；`mypy` 类型检查（可选）。
- 持续集成建议：CI 运行 `ruff check . && black --check . && pytest --maxfail=1 --disable-warnings -q`。

## 六、语音 + 小人交互逻辑
1) 用户说话 → ASR → 文本。
2) 文本 → LLM Agent：意图识别 + 信息补齐 + 决策。
3) 输出：
   - 文本反馈（含风险等级与理由）；
   - TTS 语音播放；
   - 小人动作/表情与语音同步（点头/摇头/手势）。
4) 主动提醒：天气变化、任务到期、行为预测 → 气泡 + 语音 + 动作提示。

## 七、RAG 与 Memory 整合方案
1) RAG 知识库：
   - 将生活规则与经验向量化；结合场景与实体检索；输出可引用的依据。
2) Memory：
   - 存储用户习惯/偏好（时间段、耐寒/耐热、交通方式等）；用于个性化与提醒门槛。
3) 整合流程：
   - 用户输入 → LLM 意图 → RAG 检索 → 读取 Memory → 多场景决策 → 结构化输出。

## 八、开发注意事项
1) ASR 与 TTS 需低延迟，尽量流式；
2) 小人动作与语音对齐，提供打点/时间轴；
3) RAG 规则定期更新，保留版本与数据来源；
4) Memory 涉隐私，采用最小化存储与访问控制；
5) 多源接口要有超时/重试/熔断与降级，保障鲁棒性。

## API 约定（示例）
- `POST /api/v1/query`

```json

  "query": "明早7点从浦东骑行通勤，需要带雨衣吗？",
  "locale": "zh-CN",
  "sessionId": "abc-123"
}
```

响应：

```json
{

  "entities": {"time": "2026-03-13T07:00:00+08:00", "mode": "bike", "location": {"name": "上海·浦东", "lat": 31.23, "lon": 121.53}},

  "decision": {"advice": "建议携带轻便雨衣。", "riskLevel": "medium", "reasons": ["降水概率56%","路面湿滑"]},
  "sources": ["weather-provider:forecast/v1"]
}
```

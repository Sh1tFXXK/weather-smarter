# 前后端联调说明

更新时间：2026-03-22

## 已完成
- 前端主功能已默认走真实后端接口，不再静默回退到前端 mock。
- 前端新增后端连接横幅，会明确展示当前后端地址、连接状态以及错误信息。
- 前端支持以下接入方式：
  - 直接访问 `http://127.0.0.1:8000/`
  - 从其他本地端口打开前端时，自动改连 `http://127.0.0.1:8000`
  - 直接打开 `frontend/index.html` 时，自动改连 `http://127.0.0.1:8000`
- 页面初始化会主动请求 `/api/v1/status`、`/api/v1/misc/weather`、`/api/v1/health/profile/{user_id}`。
- 以下前端功能已绑定后端：
  - 决策查询：`POST /api/v1/query`
  - 天气查询：`GET /api/v1/misc/weather`
  - 记忆读取/清理：`GET/DELETE /api/v1/memory/{session_id}`
  - 健康档案：`POST/GET/DELETE /api/v1/health/profile`
  - 健康提醒：`GET /api/v1/health/alerts/{user_id}`
  - 用户画像：`GET /api/v1/profile/analyze/{user_id}`
  - 预报：`GET /api/v1/forecast`
  - AQI：`GET /api/v1/aqi`
  - 任务创建：`POST /api/v1/tasks`
  - RAG 证据：`GET /api/v1/rag/evidence`
  - 股市影响分析：`GET /api/v1/stocks/impact`
  - 后端能力检测：`GET /api/v1/status`
  - 语音播报：前端优先调用 `POST /api/v1/tts`，失败时退回浏览器 TTS

## 当前后端真实程度
- `weather`：优先走 `backend/data/uapis_weather.py` 的上游天气代理；上游不可用时回退服务端 mock。
- `query`：一定返回后端决策结果。若 LLM 已配置，则返回 LLM 增强结果；若未配置，则返回规则引擎结果，不再直接报 503。
- `tasks`：已改为 SQLite 持久化，不再随机生成任务 ID。
- `tasks list`：已新增 `GET /api/v1/tasks`，可以读取已创建的任务。
- `stocks/impact`：已改为后端基于天气信号的规则分析，不再固定返回一份前端假数据。
- `forecast`、`aqi`：目前是“真实天气优先 + 服务端归一化 + mock 兜底”的混合实现。
- `query`、`forecast`、`aqi`、`health/alerts`：已补 `traceId` 或 `source/updatedAt` 等状态信息，便于前端识别数据来源。
- 后端已开放本机开发跨域：
  - `http://localhost:*`
  - `http://127.0.0.1:*`
  - `Origin: null`（直接打开本地 HTML 文件）

## 暂未完全完成
- 语音识别 ASR：
  - 前端仍主要使用浏览器 `SpeechRecognition`
  - 后端已有 `/api/v1/asr`，但当前前端没有完整录音上传链路
  - 若要完全切到后端，需要补 `MediaRecorder -> base64 -> /api/v1/asr`

- 天气多日/逐小时预报与 AQI：
  - 当前后端已统一为同一条接口链路返回
  - 但上游天气接口本身若未配置或不可达，会回退为服务端 mock
  - 如需“100% 外部真实数据”，需要稳定可用的 `UAPIS_BASE_URL`

- TTS / ASR 外部依赖：
  - 后端语音服务依赖 OpenAI 兼容接口
  - 需要配置 `SPEECH_PROVIDER=openai` 以及 `OPENAI_API_KEY` 或 `OPENAI_BASE_URL`

## 建议后续补齐
1. 给前端补完整的后端 ASR 录音上传链路。
2. 给 `/forecast`、`/aqi` 增加更明确的 `source`、`updatedAt` 字段到响应模型。
3. 若要完全避免任何 mock 兜底，需要部署稳定天气数据源并关闭回退逻辑。
